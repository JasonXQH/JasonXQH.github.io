5.20日记及第一次数据结构测试

第一题

1 Description

将两个有序的存有字母的链表进行合并，保证合并后的链表依然有序并去重，仅保留不重复的字母结点。

Input

n和m，分别代表两个待合并链表的长度；

n个字母，中间用空格隔开，代表第一个待合并链表；

m个字母，中间用空格隔开，代表第二个待合并链表。

Output

输出按要求合并后的链表。

Sample Input

3 3

A B D

A C D

Sample Output

A B C D

思路

第一题的思路我结合了21合并两个有序链表以及83删除链表的重复元素这两道leetcode上的简单题。

因为我们得到的是有序的字母链表，所以只要交错插入即可

拿题目给出的例子来说，现在我们把A->B->D，A->C->D都合并起来，得到了 A->A->B->C->D->D

然后我做的一步就是链表去重，原理也很简单，就是判断下一节点是否和当且节点相等，因为这是有序链表，所以不存在到处找的问题，相等的话直接删除即可。

第二题

2 Description

n个人排成一排（用链表实现）。给定一份名单，该名单中共有m个人（是n个人的子集）。现要将名单中的人进行分组，分组规则是连续排在一起的人为一组。求名单中的人一共能分成多少组？

Input

n和m，分别代表排成一排的人数和名单中的人数；

n个人（用数字代替，无重复）排成一排的顺序；

名单中包含的m个人（用数字代替，无重复）。

Output

名单中的人一共能分成多少组。

Sample Input

4 3

0 1 2 3

0 1 3

Sample Output

2

思路

这题虽然进行了包装，但是其本质，还是在于817链表组件这一道力扣题上，我的思路在之前的提接中已有提及，所以这里略去不讲

第三题

3 Description

给定一个平衡括号字符串 S，按下述规则计算该字符串的分数：

() 得 1 分。

AB 得 A + B 分，其中 A 和 B 是平衡括号字符串。例如“()()”得1+1=2分。

(A) 得 2 A 分，其中 A 是平衡括号字符串，例如“(())”得21=2分。

提示：可用栈实现，遇到左括号进栈，遇到右括号出栈后算分，算出局部分数后，可以将该分数重新压入栈中。

Input

输入包含两行；

第一行为n，代表平衡括号字符串的长度；

第二行为平衡括号字符串。

Output

输出该字符串的得分。

Sample Input1

( )

Sample Output1

1

Sample Input2

( )( )

Sample Output2

2

Sample Input3

( ( ) )

Sample Output3

2

Sample Input4

( ( ) ( ( ) ) )

Sample Output4

6

思路

这题和856括号的分数一模一样，我在5.12号写的这篇题解中有详细的思路进行阐述
